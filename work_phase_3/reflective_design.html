Accessibility Audit Report

This report identifies key accessibility issues found during testing and outlines our plan to address them.

Issue:"nameHelp" specified for the ARIA property 'aria-describedby' value is not valid

WCAG Criteria affected: WCAG 1.3.1 where information, structure, and relationships conveyed through presentation can also be programmatically determined or are available in text.

Affected Element:The 'id' "nameHelp" specified for the ARIA property 'aria-describedby' value is not valid

User Impact:People using a screen reader, including blind, low vision, neurodivergent ones and people with dexterity impairments using voice control.

Proposed Solution:By updating the ARIA property value with the unique id value of a valid visible element.

Issue: Background is less than the WCAG AA minimum requirements

WCAG Criteria:WCAG 1.4.3: Contrast Minimum

Affected Element:h3

User Impact:The ones who are unable to see color, having limited color vision, or having color blindness

Proposed Solution:Have large text that is at least 18 point or at least 14 point when bold, adjust the color of the text or its background to give at least a 3 to 1 contrast ratio

Issue:'id' "emailHelp" specified for the ARIA property 'aria-describedby' value is not valid

WCAG Criteria:1.3.1 that deals with information and relationships (Level A)

Affected Element:Input element with name, element, etc

User Impact: Ones who use a screen reader, are blind or have low vision, and neurodivergent people

Proposed Solution: Updating the ARIA property with a unique id value of a valid visible element
